\section*{Resumen}
Este documento técnico expone de manera detallada el proceso metodológico para el diseño, desarrollo e integración de librerías de software modulares en lenguajes C y C++ destinadas a microcontroladores de la familia STM32. Para ilustrar estos conceptos de manera práctica, se presenta como caso de estudio la implementación de un controlador de bajo nivel para el sensor digital de corriente, tensión y potencia INA236 a través del bus de comunicación I2C utilizando la interfaz HAL (\textit{Hardware Abstraction Layer}) de STMicroelectronics.\\

A lo largo de este informe se detalla la estructuración óptima de archivos de cabecera (\texttt{.h}) y archivos de código fuente (\texttt{.c}), las consideraciones físicas y matemáticas de la calibración interna del sensor basadas en las ecuaciones del fabricante, el manejo eficiente de variables de punto flotante en microcontroladores con núcleos ARM Cortex-M, y la configuración del entorno de compilación unificado CMake en Visual Studio Code. Este enfoque estructurado promueve la reutilización de código, facilita la depuración de hardware y garantiza una arquitectura de software embebido robusta y escalable.

\section{Fundamentos de Librerías en C para Sistemas Embebidos}

En el ámbito del diseño de firmware para sistemas embebidos, la modularidad y el encapsulamiento de funciones son indispensables para desarrollar código robusto, legible y reutilizable. Una \textbf{librería} en C se define como un conjunto de archivos independientes que empaquetan registros, estructuras de datos, macros y funciones diseñados para cumplir una tarea específica (como interactuar con un periférico de hardware o ejecutar un algoritmo de control), aislando dicha lógica del código principal de la aplicación (\texttt{main.c}).

Las ventajas clave del uso de librerías personalizadas en el desarrollo de firmware incluyen:
\begin{itemize}
	\item \textbf{Abstracción de Hardware:} El programa principal interactúa con los sensores mediante llamadas a funciones legibles (por ejemplo, \texttt{INA236\_GetCurrent\_A()}) en lugar de manipular manualmente registros del bus I2C o realizar corrimientos de bits.
	\item \textbf{Reutilización de Código:} Permite importar el mismo controlador en múltiples proyectos con diferentes microcontroladores STM32 con mínimo esfuerzo de adaptación.
	\item \textbf{Aislamiento y Mantenimiento:} Si ocurre un error en la comunicación con el periférico o se modifica el diseño físico de la placa, los cambios se limitan al código interno de la librería, sin afectar la lógica superior de la aplicación.
\end{itemize}

El diseño estándar de una librería en C consta de la separación de su estructura en dos tipos de archivos fundamentales:
\begin{description}
	\item[Archivo de Cabecera (\texttt{.h}):] Representa la \textbf{interfaz pública}. Declara qué puede hacer la librería hacia el exterior. Contiene las definiciones de registros de hardware (direcciones físicas y máscaras de bits), tipos de datos de configuración (\texttt{enum} y \texttt{struct}), y los prototipos de las funciones que el usuario puede invocar desde otros archivos del proyecto.
	\item[Archivo de Código Fuente (\texttt{.c}):] Representa la \textbf{implementación privada}. Define la lógica real de las funciones declaradas. Almacena funciones auxiliares y variables de configuración interna que están ocultas para el resto del programa, protegiendo el acceso indebido a la memoria del procesador.
\end{description}

\begin{InfoBox}{El Proceso de Compilación y Enlazado}
	Durante el desarrollo del firmware, el preprocesador reemplaza recursivamente las directivas \texttt{\#include} de los archivos \texttt{.h} en los fuentes \texttt{.c}. Posteriormente, el compilador procesa cada archivo \texttt{.c} por separado para generar archivos objeto intermedios (\texttt{.o}). Finalmente, el \textbf{Enlazador (Linker)} unifica estos objetos y resuelve las direcciones físicas de las funciones compartidas, estructurando el binario definitivo que se programará en la memoria Flash del microcontrolador.
\end{InfoBox}

\section{Estructura Estándar de Archivos Header (\texttt{.h}) y Source (\texttt{.c})}

Para garantizar la portabilidad y evitar conflictos de compilación en proyectos de STM32 bajo el ecosistema de CMake, se deben seguir estrictamente ciertos patrones de diseño al redactar archivos \texttt{.h} y \texttt{.c}.

\subsection{Estructura de un Archivo de Cabecera (\texttt{.h})}
Un archivo de cabecera profesional debe contener los siguientes elementos ordenados de manera lógica:

\begin{enumerate}
	\item \textbf{Guardas de Inclusión:} Evitan errores de doble declaración de tipos en el enlazador si el archivo es importado por múltiples dependencias. Se estructuran mediante las directivas \texttt{\#ifndef}, \texttt{\#define} y \texttt{\#endif}.
	\item \textbf{Inclusión de Dependencias:} Archivos del sistema como \texttt{stdint.h} (para tipos de enteros con tamaño fijo como \texttt{uint16\_t}) y las librerías HAL específicas del microcontrolador (por ejemplo, \texttt{stm32f3xx\_hal.h} o la cabecera general \texttt{main.h}).
	\item \textbf{Definiciones de Constantes (Macros):} Direcciones físicas de registros I2C de lectura/escritura y máscaras para configurar registros de control a nivel de bit.
	\item \textbf{Tipos de Datos Enumerados (\texttt{enum}):} Declaraciones de tipos con nombre que representan los valores admitidos de configuración (por ejemplo, promedios, tiempos de conversión y modos de operación).
	\item \textbf{Estructura de Control (\texttt{HandleTypeDef}):} Una estructura en C que almacena las variables de estado e instancias físicas del periférico. Es la firma digital de la librería y vincula el controlador a un bus I2C real.
	\item \textbf{Prototipos de Funciones Públicas:} Declaración formal de las funciones expuestas de la librería.
\end{enumerate}

\subsection{Estructura de un Archivo de Código Fuente (\texttt{.c})}
El archivo de implementación física organiza el código de la siguiente manera:

\begin{enumerate}
	\item \textbf{Inclusión del Header Correspondiente:} Permite que el archivo compile conociendo todos los tipos y macros declarados en su cabecera (por ejemplo, \texttt{\#include "INA236.h"}).
	\item \textbf{Prototipos de Funciones Estáticas Privadas:} Declaraciones de funciones locales a las que solo se puede acceder dentro del mismo archivo fuente.
	\item \textbf{Implementación de Funciones Auxiliares Privadas (\texttt{static}):} Funciones de soporte lógico interno.
	\item \textbf{Implementación de Funciones del API Público:} Bloque con el desarrollo del código real para inicialización, lectura y control físico del sensor.
\end{enumerate}

\begin{WarningBox}{La Importancia del Modificador \texttt{static}}
	Al declarar una función o variable global como \texttt{static} en el archivo \texttt{.c}, se le indica al compilador que restrinja su visibilidad exclusivamente a ese módulo. Esto previene conflictos de nombres si otra librería utiliza un nombre idéntico, y permite que el optimizador del compilador genere llamadas a función en línea (\textit{inline}) mucho más rápidas y eficientes en ensamblador.
\end{WarningBox}

\section{Análisis Detallado del Archivo de Cabecera (\texttt{INA236.h})}

El archivo de cabecera \texttt{INA236.h} constituye la interfaz pública de la librería. En él se definen las constantes de dirección I2C, el mapa de registros físicos del sensor, las máscaras y posiciones de bits para configuraciones internas, los tipos de datos enumerados de control, la estructura principal de manejo del dispositivo y los prototipos de las funciones de la API.

\subsection{Constantes y Macros de Utilidad}
Al inicio del archivo se configuran los parámetros del bus I2C y una macro de ayuda para evaluación de bits:
\begin{lstlisting}[style=CMakeStyle]
#define INA236_DEFAULT_ADDRESS          0x40 
#define INA236_TRIALS                   5     
#define CHECK_BIT(var,pos) ((var) & (1<<(pos)))
\end{lstlisting}

\begin{itemize}
	\item \texttt{INA236\_DEFAULT\_ADDRESS} (\texttt{0x40}): Dirección I2C predeterminada de 7 bits del integrado cuando las entradas físicas de dirección A0 y A1 se conectan a la señal de GND.
	\item \texttt{INA236\_TRIALS} (\texttt{5}): Número de reintentos máximos de consulta de presencia que el driver ejecutará en el bus I2C antes de declarar un error de desconexión analógica.
	\item \texttt{CHECK\_BIT(var, pos)}: Macro que evalúa de forma rápida el estado lógico de un bit específico en una variable de control, devolviendo un valor distinto de cero si el bit en la posición \texttt{pos} está en nivel alto (\texttt{1}):
		\begin{equation}
			\text{CHECK\_BIT}(var, pos) = var \ \& \ (1 \ll pos)
		\end{equation}
\end{itemize}

\subsection{Definición del Mapa de Registros}
Las direcciones físicas de los registros del INA236 se declaran como macros de preprocesador para asociar nombres lógicos legibles con sus posiciones reales de memoria en el sensor:
\begin{lstlisting}[style=CMakeStyle]
/* INA236 Register Map Addresses */
#define INA236_CONFIGURATION_REGISTER 0x00 /* Configuration Register - Reset Value = 4127h*/
#define INA236_SHUNT_VOLTAGE_REGISTER 0x01 /* Shunt Voltage Register - Reset Value = 0000h*/
#define INA236_BUS_VOLTAGE_REGISTER   0x02 /* Bus Voltage Register   - Reset Value = 0000h*/
#define INA236_POWER_REGISTER         0x03 /* Power Register         - Reset Value = 0000h*/
#define INA236_CURRENT_REGISTER       0x04 /* Current Register       - Reset Value = 0000h*/
#define INA236_CALIBRATION_REGISTER   0x05 /* Calibration Register   - Reset Value = 0000h*/
#define INA236_MASK_ENABLE_REGISTER   0x06 /* Mask/Enable Register   - Reset Value = 0000h*/
#define INA236_ALERT_LIMIT_REGISTER   0x07 /* Alert Limit Register   - Reset Value = 0000h*/
#define MANUFACTURER_ID_REGISTER      0x3E /* Device ID Register     - Reset Value = 5449h*/  
#define DEVICE_ID_REGISTER            0x3F /* Manufacturer ID Register-Reset Value = A080h*/
\end{lstlisting}

\begin{itemize}
	\item \texttt{INA236\_CONFIGURATION\_REGISTER} (\texttt{0x00}): Registro de 16 bits para programar resets, rangos, promedios y tiempos de conversión.
	\item \texttt{INA236\_SHUNT\_VOLTAGE\_REGISTER} (\texttt{0x01}): Almacena la tensión diferencial en bornes del shunt.
	\item \texttt{INA236\_BUS\_VOLTAGE\_REGISTER} (\texttt{0x02}): Almacena la tensión del bus común.
	\item \texttt{INA236\_POWER\_REGISTER} (\texttt{0x03}): Almacena el resultado del cálculo interno de potencia.
	\item \texttt{INA236\_CURRENT\_REGISTER} (\texttt{0x04}): Almacena el cálculo interno de corriente.
	\item \texttt{INA236\_CALIBRATION\_REGISTER} (\texttt{0x05}): Almacena el coeficiente de calibración de escala.
	\item \texttt{INA236\_MASK\_ENABLE\_REGISTER} (\texttt{0x06}): Registro para configurar alertas e interrupciones.
	\item \texttt{INA236\_ALERT\_LIMIT\_REGISTER} (\texttt{0x07}): Almacena el límite físico de comparación de la alerta.
	\item \texttt{MANUFACTURER\_ID\_REGISTER} (\texttt{0x3E}) y \texttt{DEVICE\_ID\_REGISTER} (\texttt{0x3F}): Registros de solo lectura que almacenan las firmas de silicio de fábrica.
\end{itemize}

\subsection{Máscaras y Posiciones de Bits}
Para realizar operaciones de bits seguras y eficientes de lectura-modificación-escritura en los registros de control, se implementa un diseño estándar basado en dos macros por cada campo de bits:
\begin{itemize}
	\item \textbf{Posición (\texttt{\_Pos}):} Define el bit menos significativo o bit inicial del campo de configuración en la palabra de 16 bits (por ejemplo, el campo de promedio \texttt{AVG} inicia en el bit 9: \texttt{\#define INA236\_AVG\_Pos (9U)}).
	\item \textbf{Máscara (\texttt{\_Mask}):} Define la máscara lógica de bits activos que cubre todo el campo. Se genera desplazando el valor máximo del campo por su posición (por ejemplo: \texttt{\#define INA236\_AVG\_Mask (0x7U $\ll$ INA236\_AVG\_Pos)}).
\end{itemize}
Este diseño se aplica en el registro de configuración para los campos de modo (\texttt{MODE}), tiempo de conversión del shunt (\texttt{VSHCT}), tiempo del bus (\texttt{VBUSCT}), promedio (\texttt{AVG}), rango del ADC (\texttt{ADCRANGE}) y reinicio por software (\texttt{RST}).
De igual forma, en el registro de alertas \texttt{MASK\_ENABLE} se definen las máscaras individuales de 1 bit para evaluar banderas como conmutación de alerta por conversión lista (\texttt{CNVR}), alerta por sobrepotencia (\texttt{POL}), caída de tensión en bus (\texttt{BUL}), sobretensión en bus (\texttt{BOL}), caída en shunt (\texttt{SUL}) o sobretensión en shunt (\texttt{SOL}), además de la polaridad de alerta (\texttt{APOL}) y la bandera de desbordamiento matemático (\texttt{OVF}).

\section{El Sensor de Monitoreo de Corriente y Potencia INA236}

El \textbf{INA236} es un monitor digital de corriente bidireccional y tensión de bus de 16 bits de resolución, equipado con una interfaz compatible con buses I2C y SMBus. Es ampliamente utilizado en el diseño de fuentes de alimentación inteligentes, protección de sobrecorriente para motores y telemetría de energía en tarjetas de desarrollo embebidas.

\subsection{Especificaciones Técnicas Clave}
\begin{itemize}
	\item \textbf{Tensión de Alimentación:} Opera en un rango de 1.7 V a 5.5 V.
	\item \textbf{Rango del Modo Común del Bus (VBUS):} Mide tensiones en la carga desde 0 V hasta 48 V de forma independiente de la alimentación.
	\item \textbf{Resolución del ADC Interno:} Conversor analógico a digital de 16 bits de ultra bajo ruido.
	\item \textbf{Rangos de Medición de Shunt Seleccionables:} Rango de entrada de $\pm 81.92\text{ mV}$ (para grandes corrientes) y $\pm 20.48\text{ mV}$ (para optimizar la resolución en corrientes pequeñas).
	\item \textbf{Parámetros de Calibración Interna:} Cuenta con un multiplicador matemático por hardware que calcula y devuelve de forma directa la lectura de corriente en Amperes e incluso la potencia del sistema en Watts tras configurarse su registro de calibración.
	\item \textbf{Entrada de Alerta Configurable:} Pin físico ALERT de drenador abierto para eventos de sobre/subtensión de bus, sobre/subcorriente o desbordamiento matemático.
\end{itemize}

\subsection{Mapa de Registros Internos}

\subsection{Matemáticas de Calibración del Sensor}
Para que el procesador interno del INA236 calcule directamente los Amperes y los Watts sin cargar de trabajo al CPU del microcontrolador STM32, se debe programar el registro \texttt{CALIBRATION} ($SHUNT\_CAL$) según las siguientes directrices matemáticas del fabricante:

\subsubsection{Cálculo del $Current\_LSB$ Mínimo}
Primero, se determina la resolución de paso de corriente más pequeña que se puede medir físicamente con el convertidor de 16 bits (donde $2^{15} = 32768$ valores positivos posibles en modo bipolar):
\begin{equation}
	Current\_LSB_{\text{min}} = \frac{I_{\text{max\_esperada}}}{32768}
\end{equation}

Una vez conocido el límite físico absoluto, se selecciona un valor redondeado de $Current\_LSB$ que sea un número entero o decimal "amigable" y limpio en pasos de $1$, $2$ o $5$ veces una potencia de 10 (por ejemplo, $10\,\mu\text{A}$, $20\,\mu\text{A}$ o $50\,\mu\text{A}$), garantizando que cumpla con la restricción de diseño:
\begin{equation}
	Current\_LSB_{\text{min}} \le Current\_LSB < 2.5 \times Current\_LSB_{\text{min}}
\end{equation}

\subsubsection{Cálculo del Registro $SHUNT\_CAL$}
A partir del $Current\_LSB$ seleccionado y del valor resistivo de la resistencia del shunt ($R_{\text{shunt}}$ en Ohms), se calcula el coeficiente entero a escribir en el registro de calibración del chip utilizando la constante de escala fija del fabricante ($0.00512$):
\begin{equation}
	SHUNT\_CAL = \frac{0.00512}{Current\_LSB \times R_{\text{shunt}}}
\end{equation}

\begin{WarningBox}{Escalado por Rango de Tensión del ADC}
	El valor base de la ecuación anterior asume una configuración de rango analógico de entrada de $\pm 81.92\text{ mV}$ (\texttt{ADCRANGE = 0}). Si se activa el modo de alta resolución de $\pm 20.48\text{ mV}$ (\texttt{ADCRANGE = 1}), el valor obtenido en la fórmula del registro $SHUNT\_CAL$ \textbf{debe dividirse obligatoriamente entre 4} antes de programarse en el sensor.
\end{WarningBox}

\subsubsection{Fórmulas de Conversión a Magnitudes Físicas en STM32}
Una vez que el registro de calibración ha sido programado con éxito, los valores obtenidos de los registros se convierten a unidades del Sistema Internacional mediante las siguientes relaciones:
\begin{itemize}
	\item \textbf{Tensión del Shunt ($V_{\text{shunt}}$):} Multiplica la lectura directa en dos de dos de dos de dos complementos por la resolución del ADC.
	\begin{equation}
		V_{\text{shunt}}\text{ [mV]} = Raw\_Shunt \times \text{Resolución (V)} \times 1000.0
	\end{equation}
	Donde la resolución es de $2.5\,\mu\text{V/LSB}$ para el rango de $81.92\text{ mV}$, y de $0.625\,\mu\text{V/LSB}$ para el rango de $20.48\text{ mV}$.
	\item \textbf{Tensión de Bus ($V_{\text{bus}}$):} Tiene un paso de tensión de entrada fijo de $1.6\text{ mV/LSB}$ ($0.0016\text{ V}$):
	\begin{equation}
		V_{\text{bus}}\text{ [V]} = Raw\_Bus \times 0.0016
	\end{equation}
	\item \textbf{Corriente ($I$):} Multiplica directamente el registro firmado por el LSB físico configurado:
	\begin{equation}
		I\text{ [A]} = Raw\_Current \times Current\_LSB
	\end{equation}
	\item \textbf{Potencia ($P$):} La potencia se calcula en base a la corriente y la tensión, y está acoplada al LSB por una constante de escala de $32$ veces el LSB de corriente:
	\begin{equation}
		P\text{ [W]} = Raw\_Power \times \left(32.0 \times Current\_LSB\right)
	\end{equation}
\end{itemize}

\subsection{El Problema de Lecturas Nulas en Potencia ante Bajas Magnitudes}
Un comportamiento reportado de forma común en el diseño de controladores para el INA236 es que, ante voltajes de bus o corrientes de carga muy pequeñas (por ejemplo, corriente de $11.7\text{ mA}$ y caída de tensión del shunt de apenas unos microvoltios), el registro físico de potencia \texttt{POWER} lee un valor absoluto de cero, a pesar de que el registro de corriente sí reporta lecturas válidas. 

Esto se debe al diseño interno de cálculo matemático de punto fijo del chip. El coprocesador interno del INA236 realiza la multiplicación directa de los registros crudos de corriente y tensión, aplicando una escala rígida en enteros:
\begin{equation}
	POWER_{\text{raw}} = \frac{CURRENT_{\text{raw}} \times VBUS_{\text{raw}}}{20000}
\end{equation}

Si el producto entre $CURRENT_{\text{raw}}$ y $VBUS_{\text{raw}}$ es menor a $20000$, la división entera trunca el resultado a $0$. Para solventar esta limitación física en silicio y medir con alta fidelidad, existen dos alternativas de diseño:
\begin{enumerate}
	\item \textbf{Reducir el $Current\_LSB$:} Configurar un LSB de corriente más pequeño (por ejemplo, bajando de $1\text{ mA}$ a $100\,\mu\text{A}$), lo cual incrementará el valor del registro entero $CURRENT_{\text{raw}}$ para la misma corriente física, evitando el truncamiento a cero.
	\item \textbf{Cálculo de Potencia por Software:} Si la corriente y la tensión del bus son extremadamente bajas y no se desea modificar la escala de hardware, se puede implementar el cálculo de la potencia en punto flotante directamente en el procesador del STM32 mediante software:
	\begin{equation}
		P\text{ [W]} = I\text{ [A]} \times V_{\text{bus}}\text{ [V]}
	\end{equation}
\end{enumerate}

\subsection{Funciones de Comunicación de Bajo Nivel}
Estas funciones implementan la interfaz directa con la capa física del protocolo I2C. Su principal tarea es realizar la traducción de formato entre el protocolo de bus Big-Endian (donde se transmite el byte más significativo primero en la red) y la arquitectura interna Little-Endian del núcleo ARM Cortex-M del microcontrolador STM32.

%--- WRITE REGISTER ---
\subsubsection{Función \texttt{INA236\_WriteRegister}}
Escribe un valor de control o configuración de 16 bits en una dirección física de registro en el silicio del sensor.

\begin{itemize}
	\item \textbf{Parámetros de Entrada:}
		\begin{description}
			\item[\texttt{ina236}:] Puntero a la estructura de control de tipo \texttt{INA236\_HandleTypeDef}. Utiliza el miembro \texttt{hi2c} para obtener el manejador físico del bus I2C de la HAL, y el miembro \texttt{\_devAddress} para conocer la dirección esclava de 7 bits del chip.
			\item[\texttt{registerAddress}:] Dirección de 8 bits del registro destino en el INA236 (por ejemplo, \texttt{0x00} para configuración o \texttt{0x05} para calibración).
			\item[\texttt{value}:] Entero sin signo de 16 bits que contiene los datos que se grabarán en el registro seleccionado.
		\end{description}
	\item \textbf{Retorno:} Código de estado de tipo \texttt{HAL\_StatusTypeDef}. Devuelve \texttt{HAL\_OK} si la escritura física se concreta con éxito o \texttt{HAL\_ERROR} si el sensor no responde en el bus.
	\item \textbf{Funcionamiento Interno:}
		La función realiza una conversión de formato dividiendo el dato de 16 bits en dos bytes separados:
		\begin{equation}
			\text{address}[0] = (value \gg 8) \ \& \ \text{0xFF} \quad (\text{Byte de peso alto / MSB})
		\end{equation}
		\begin{equation}
			\text{address}[1] = (value \gg 0) \ \& \ \text{0xFF} \quad (\text{Byte de peso bajo / LSB})
		\end{equation}
		Posteriormente, verifica la presencia física del dispositivo en la dirección esclava del bus I2C mediante una llamada de preparación \texttt{HAL\_I2C\_IsDeviceReady}, enviando la dirección de 7 bits desplazada un bit a la izquierda (\texttt{devAddress $\ll$ 1}) para adecuar el formato de direccionamiento de 8 bits de la HAL de ST. Si el chip responde antes del tiempo de expiración (\textit{Timeout}), se ejecuta la función de escritura directa de memoria de la HAL \texttt{HAL\_I2C\_Mem\_Write} en modo de tamaño de dirección de registro de 8 bits (\texttt{I2C\_MEMADD\_SIZE\_8BIT}) y tamaño de transmisión de búfer de 16 bits (\texttt{I2C\_MEMADD\_SIZE\_16BIT}).
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
HAL_StatusTypeDef INA236_WriteRegister(INA236_HandleTypeDef *ina236, uint8_t registerAddress, uint16_t value)
{
  uint8_t address[2];
  HAL_StatusTypeDef isDeviceReady;
  // Split the 16-bit value into two bytes (MSB first)
  address[0] = (value >> 8) & 0xFF;
  address[1] = (value >> 0) & 0xFF;
  
  // Check if the device is ready on the I2C bus
  isDeviceReady = HAL_I2C_IsDeviceReady(ina236->hi2c, (ina236->_devAddress) << 1, INA236_TRIALS, HAL_MAX_DELAY);
  if (isDeviceReady == HAL_OK)
  {
    // Write the 16-bit register to the device
    if (HAL_I2C_Mem_Write(ina236->hi2c, (ina236->_devAddress) << 1, registerAddress, I2C_MEMADD_SIZE_8BIT, (uint8_t*)address, I2C_MEMADD_SIZE_16BIT, HAL_MAX_DELAY) == HAL_OK)
    {
      return HAL_OK;
    }
  }
  return HAL_ERROR;
}
\end{lstlisting}

%--- READ REGISTER ---
\subsubsection{Función \texttt{INA236\_ReadRegister}}
Lee los datos de 16 bits contenidos en un registro específico del mapa de memoria del INA236.

\begin{itemize}
	\item \textbf{Parámetros de Entrada:}
		\begin{description}
			\item[\texttt{ina236}:] Puntero a la estructura \texttt{INA236\_HandleTypeDef} para referenciar el bus I2C y la dirección del sensor.
			\item[\texttt{registerAddress}:] Dirección física de 8 bits del registro al que se desea acceder para su lectura (por ejemplo, \texttt{0x01} para voltaje de shunt o \texttt{0x04} para corriente).
		\end{description}
	\item \textbf{Retorno:} Entero de 16 bits sin signo con los datos leídos del registro. En caso de que ocurra una falla física de comunicación I2C o el dispositivo no esté presente en el bus, la función retorna el valor límite de control \texttt{0xFFFF}.
	\item \textbf{Funcionamiento Interno:}
		La función valida la presencia del chip esclavo en el bus I2C mediante la directiva \texttt{HAL\_I2C\_IsDeviceReady} (desplazando la dirección \texttt{\_devAddress $\ll$ 1}). Si el estado de la comunicación es correcto, invoca a la función de lectura de memoria \texttt{HAL\_I2C\_Mem\_Read}. Esta lectura extrae los datos de 16 bits del sensor recibiéndolos en un búfer temporal de dos bytes de tamaño en formato Big-Endian. Finalmente, realiza una operación de desplazamiento y suma lógica OR para unir los bytes individuales en un entero nativo Little-Endian de 16 bits:
		\begin{equation}
			\text{Retorno} = (\text{registerResponse}[0] \ll 8) \ | \ \text{registerResponse}[1]
		\end{equation}
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
uint16_t INA236_ReadRegister(INA236_HandleTypeDef *ina236, uint8_t registerAddress)
{
  uint8_t registerResponse[2] = {0}; 
  HAL_StatusTypeDef isDeviceReady;   

  // Check if the device is ready on the I2C bus
  isDeviceReady = HAL_I2C_IsDeviceReady(ina236->hi2c, (ina236->_devAddress) << 1, INA236_TRIALS, HAL_MAX_DELAY);

  if (isDeviceReady == HAL_OK)
  {
    // Read the 2-byte register data from the device
    if (HAL_I2C_Mem_Read(ina236->hi2c, (ina236->_devAddress) << 1, registerAddress, I2C_MEMADD_SIZE_8BIT, registerResponse, sizeof(registerResponse), HAL_MAX_DELAY) == HAL_OK)
    {
        // Combine MSB and LSB to return the 16-bit value
        return (uint16_t)((registerResponse[0] << 8) | registerResponse[1]);
    }    
  }
  // Returns a maximum control value (0xFFFF) indicating a communication error
  return 0xFFFF;
}
\end{lstlisting}

%--- ROUND CURRENT LSB ---
\subsubsection{Función \texttt{INA236\_RoundCurrentLsb} (Auxiliar Privada)}
Aplica reglas matemáticas logarítmicas para redondear el paso mínimo del LSB de corriente obtenido a un valor decimal limpio de la serie de paso $1$, $2$ o $5$.

\begin{itemize}
	\item \textbf{Parámetros de Entrada:}
		\begin{description}
			\item[\texttt{lsbMin}:] El LSB mínimo en Amperes por paso obtenido de la división de la corriente máxima de diseño entre la resolución del convertidor analógico a digital.
		\end{description}
	\item \textbf{Retorno:} Valor de tipo \texttt{double} con el LSB redondeado definitivo.
	\item \textbf{Funcionamiento Interno:}
		La función obtiene la potencia de 10 inferior más cercana mediante el logaritmo base 10 del LSB mínimo:
		\begin{equation}
			\text{logLsb} = \log_{10}(\text{lsbMin})
		\end{equation}
		\begin{equation}
			\text{powerOf10} = 10^{\lfloor\text{logLsb}\rfloor}
		\end{equation}
		Luego, normaliza el valor a un número real contenido entre $1.0$ y $10.0$ mediante:
		\begin{equation}
			\text{normalized} = \frac{\text{lsbMin}}{\text{powerOf10}}
		\end{equation}
		Finalmente, evalúa el valor normalizado en un bloque de toma de decisiones condicionales para aproximarlo al paso inmediato superior en la serie $1$, $2$, $5$ o $10$, multiplicándolo nuevamente por la potencia de 10 de escala para restaurar el orden de magnitud decimal.
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
static double INA236_RoundCurrentLsb(double lsbMin)
{
    // Find power of 10 below lsbMin
    double logLsb = log10(lsbMin);
    double powerOf10 = pow(10, floor(logLsb));
    
    // Normalize to a value between 1.0 and 10.0
    double normalized = lsbMin / powerOf10;
    double roundedLsb;
    
    // Round up to the nearest 1, 2, or 5 step
    if (normalized <= 1.0)
    {
        roundedLsb = 1.0 * powerOf10;
    }
    else if (normalized <= 2.0)
    {
        roundedLsb = 2.0 * powerOf10;
    }
    else if (normalized <= 5.0)
    {
        roundedLsb = 5.0 * powerOf10;
    }
    else
    {
        roundedLsb = 10.0 * powerOf10;
    }
    
    return roundedLsb;
}
\end{lstlisting}

%--- SET CALIBRATION ---
\subsubsection{Función \texttt{INA236\_SetCalibration}}
Configura el valor resistivo del shunt y calcula e inyecta el coeficiente numérico de calibración de escala analógica en el registro respectivo del silicio del sensor.

\begin{itemize}
	\item \textbf{Parámetros de Entrada:}
		\begin{description}
			\item[\texttt{ina236}:] Puntero a la estructura del driver.
			\item[\texttt{rShuntValue}:] Resistencia física del shunt soldado en la placa de hardware (expresada en Ohms).
			\item[\texttt{maxCurrent}:] Máxima corriente de carga que se espera circule por la línea (expresada en Amperes).
		\end{description}
	\item \textbf{Retorno:} Ninguno (\texttt{void}).
	\item \textbf{Funcionamiento Interno:}
		La función almacena localmente los parámetros de entrada y calcula el LSB de corriente absoluto de hardware dividiendo la corriente máxima esperada entre el rango analógico disponible en 15 bits ($32768$). 
		Posteriormente, invoca a la función auxiliar privada \texttt{INA236\_RoundCurrentLsb} para obtener un paso numérico amigable y limpio, salvando dicho valor en el miembro \texttt{\_currentLsb}.
		A continuación, se computa el valor entero del registro de calibración aplicando la ecuación teórica del fabricante:
		\begin{equation}
			SHUNT\_CAL = \frac{0.00512}{\text{roundedLsb} \times rShuntValue}
		\end{equation}
		La función evalúa el estado del bit del rango dinámico del ADC. Si \texttt{\_adcRange == 1} (que corresponde al rango analógico estrecho de entrada de $\pm 20.48\text{ mV}$), divide el valor obtenido de la ecuación por 4 de forma obligatoria para ajustar la escala del multiplicador físico interno del chip. Finalmente, graba el coeficiente final en el registro \texttt{INA236\_CALIBRATION\_REGISTER} (\texttt{0x05}) llamando a \texttt{INA236\_WriteRegister}.
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
void INA236_SetCalibration(INA236_HandleTypeDef *ina236, double rShuntValue, int maxCurrent)
{
    ina236->_shuntResistor = rShuntValue;
    ina236->_maximumCurrent = (double)maxCurrent;

    // Calculate the minimum Current LSB: Max_Current / 2^15 (2^15 = 32768)
    double currentLsbMinimum = (double)maxCurrent / 32768.0; 
    ina236->_currentLsbMin = currentLsbMinimum;

    // Auto-calculate a clean, round LSB (always satisfies: lsbMin <= lsb < 2.5 * lsbMin)
    double roundedLsb = INA236_RoundCurrentLsb(currentLsbMinimum);
    ina236->_currentLsb = roundedLsb;

    // Calculate the Calibration register value: 0.00512 / (roundedLsb * rShuntValue)
    uint16_t shuntCal = 0.00512 / (roundedLsb * rShuntValue);

    // According to datasheet, SHUNT_CAL must be divided by 4 for ADCRANGE = 1 (20.48 mV)
    if (ina236->_adcRange == 1)
    {
        shuntCal = shuntCal / 4;
    }

    INA236_WriteRegister(ina236, INA236_CALIBRATION_REGISTER, shuntCal);
}
\end{lstlisting}

%--- RESET DEVICE ---
\subsubsection{Función \texttt{INA236\_ResetDevice}}
Fuerza un reinicio de hardware en caliente por software sobre todos los registros internos de control del INA236.

\begin{itemize}
	\item \textbf{Parámetros de Entrada:}
		\begin{description}
			\item[\texttt{ina236}:] Puntero a la estructura del driver.
		\end{description}
	\item \textbf{Retorno:} Ninguno.
	\item \textbf{Funcionamiento Interno:}
		La función escribe de forma directa la máscara de control \texttt{INA236\_RST\_Mask} (\texttt{0x8000}, correspondiente al bit 15 de control del registro de configuración \texttt{0x00}). Al activarse este bit, el procesador interno del chip borra toda su memoria volátil restaurando los parámetros predeterminados de fábrica. El bit se limpia de manera automática en el silicio.
		La función introduce una espera activa por hardware de $2\text{ ms}$ mediante \texttt{HAL\_Delay} para garantizar la correcta estabilización del bus I2C y actualiza la bandera interna local del rango del ADC a su estado básico de $\pm 81.92\text{ mV}$.
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
void INA236_ResetDevice(INA236_HandleTypeDef *ina236)
{
    // Write directly the reset bit (bit 15 = 0x8000) to CONFIGURATION
    INA236_WriteRegister(ina236, INA236_CONFIGURATION_REGISTER, INA236_RST_Mask);
    
    // Wait a brief delay for internal circuits to settle
    HAL_Delay(2); 
    
    // After reset, device returns to standard range +/- 81.92 mV
    ina236->_adcRange = INA236_ADC_RANGE_81_92_MV;
}
\end{lstlisting}

\subsection{Funciones de Configuración de Periféricos}
Estas funciones permiten modificar de manera selectiva y segura campos específicos de bits contenidos en el registro \texttt{CONFIGURATION} (\texttt{0x00}), evitando alterar los valores configurados en otros campos adyacentes del mismo registro.

%--- SET MODE ---
\subsubsection{Función \texttt{INA236\_SetMode}}
Modifica el modo de operación del convertidor analógico a digital del sensor.

\begin{itemize}
	\item \textbf{Parámetros de Entrada:}
		\begin{description}
			\item[\texttt{ina236}:] Puntero a la estructura del driver.
			\item[\texttt{mode}:] Modo seleccionado de tipo enumerado \texttt{INA236\_Mode\_TypeDef} (por ejemplo, continuo, apagado o disparo único).
		\end{description}
	\item \textbf{Funcionamiento Interno:}
		Aplica la técnica de lectura-modificación-escritura:
		\begin{enumerate}
			\item Lee el registro de configuración de 16 bits llamando a \texttt{INA236\_ReadRegister}.
			\item Limpia los bits del modo (bits 0, 1 y 2) aplicando una operación lógica AND con la máscara del modo invertida:
				\begin{equation}
					\text{regValue} = \text{regValue} \ \& \ (\sim\text{INA236\_MODE\_Mask})
				\end{equation}
			\item Carga el nuevo modo de operación alineando los bits mediante un desplazamiento a su posición y protegiéndolos mediante un AND lógico con la máscara antes de sumarlos con un OR lógico:
				\begin{equation}
					\text{regValue} = \text{regValue} \ | \ ((\text{mode} \ll \text{INA236\_MODE\_Pos}) \ \& \ \text{INA236\_MODE\_Mask})
				\end{equation}
			\item Escribe el resultado final de vuelta en el sensor mediante \texttt{INA236\_WriteRegister}.
		\end{enumerate}
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
void INA236_SetMode(INA236_HandleTypeDef *ina236, INA236_Mode_TypeDef mode)
{
    uint16_t regValue = INA236_ReadRegister(ina236, INA236_CONFIGURATION_REGISTER);
    regValue &= ~INA236_MODE_Mask;
    regValue |= (mode << INA236_MODE_Pos) & INA236_MODE_Mask;
    INA236_WriteRegister(ina236, INA236_CONFIGURATION_REGISTER, regValue);
}
\end{lstlisting}

%--- SET ADC RANGE ---
\subsubsection{Función \texttt{INA236\_SetAdcRange}}
Configura el rango dinámico de tensión en la entrada analógica del convertidor diferencial de shunt y actualiza las constantes de resolución matemática del firmware.

\begin{itemize}
	\item \textbf{Parámetros de Entrada:}
		\begin{description}
			\item[\texttt{ina236}:] Puntero a la estructura del driver.
			\item[\texttt{range}:] Rango de entrada física del ADC de la enumeración \texttt{INA236\_AdcRange\_TypeDef} (\texttt{0} para $\pm 81.92\text{ mV}$ o \texttt{1} para $\pm 20.48\text{ mV}$).
		\end{description}
	\item \textbf{Funcionamiento Interno:}
		El bit 12 de configuración es aislado y modificado mediante la máscara \texttt{INA236\_ADCRANGE\_Mask} (\texttt{0x1000}) y la posición \texttt{INA236\_ADCRANGE\_Pos} (\texttt{12U}).
		Adicionalmente, actualiza el valor lógico \texttt{\_adcRange} de la estructura de datos local y recalcula la resolución de paso analógico a digital (\texttt{\_resolution}) y el límite del rango dinámico (\texttt{\_shuntAdcRange}) que serán usados por las funciones de cálculo de punto flotante en el microcontrolador:
		\begin{itemize}
			\item Si se selecciona el rango estrecho de $\pm 20.48\text{ mV}$, la resolución física del ADC se fija en $625\text{ nV/LSB}$ (\texttt{0.000000625}).
			\item Si se selecciona el rango ancho de $\pm 81.92\text{ mV}$, la resolución física se establece en $2.5\,\mu\text{V/LSB}$ (\texttt{0.0000025}).
		\end{itemize}
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
void INA236_SetAdcRange(INA236_HandleTypeDef *ina236, INA236_AdcRange_TypeDef range)
{

    regValue &= ~INA236_ADCRANGE_Mask;
    regValue |= (range << INA236_ADCRANGE_Pos) & INA236_ADCRANGE_Mask;
    INA236_WriteRegister(ina236, INA236_CONFIGURATION_REGISTER, regValue);
    
    ina236->_adcRange = (_Bool)range;
    if (range == INA236_ADC_RANGE20_48_MV)
    {
        ina236->_shuntAdcRange = 0.02048;
        ina236->_resolution = 0.000000625; // 625 nV/LSB
    }
    else
    {
        ina236->_shuntAdcRange = 0.08192;
        ina236->_resolution = 0.0000025;   // 2.5 uV/LSB
    }
}
\end{lstlisting}

\subsection{Funciones de Gestión de Alertas por Hardware}

%--- SET/GET ALERT POLARITY ---
\subsubsection{Configuración de Polaridad del Pin ALERT}
Define la lógica eléctrica del pin físico ALERT del sensor en modo normalmente abierto activo bajo u opcionalmente activo alto.

\begin{itemize}
	\item \textbf{Funcionamiento Interno:}
		La función de escritura manipula el bit 1 (\texttt{APOL}) del registro de máscara mediante \texttt{INA236\_APOL\_Mask}.
		La función de lectura lee el registro esclavo, verifica si no ocurrió un error I2C (en cuyo caso retorna la polaridad segura por defecto de activo bajo) y extrae el estado físico realizando una operación lógica AND y un desplazamiento a la derecha para obtener un valor limpio de 0 o 1 que coincide con las definiciones de la enumeración:
		\begin{equation}
			\text{Retorno} = \frac{\text{regValue} \ \& \ \text{INA236\_APOL}}{\text{INA236\_APOL\_Pos}}
		\end{equation}
		La función además imprime un diagnóstico legible en español mediante UART a través de la consola de depuración del sistema utilizando la función estándar \texttt{printf}.
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
void INA236_SetAlertPolarity(INA236_HandleTypeDef *ina236, INA236_AlertPol_TypeDef polarity)
{
    uint16_t regValue = INA236_ReadRegister(ina236, INA236_MASK_ENABLE_REGISTER);
    regValue &= ~INA236_APOL_Mask;
    regValue |= (polarity << INA236_APOL_Pos) & INA236_APOL_Mask;
    INA236_WriteRegister(ina236, INA236_MASK_ENABLE_REGISTER, regValue);
}

INA236_AlertPol_TypeDef INA236_GetAlertPolarity(INA236_HandleTypeDef *ina236)
{
    uint16_t regValue = INA236_ReadRegister(ina236, INA236_MASK_ENABLE_REGISTER);
    if (regValue == 0xFFFF)
    {
        return INA236_ACTIVE_LOW;
    }
    INA236_AlertPol_TypeDef polarity = (INA236_AlertPol_TypeDef)((regValue & INA236_APOL) >> INA236_APOL_Pos);
    if (polarity == INA236_ACTIVE_HIGH)
    {
        printf("Alert pin is configured as ACTIVE-HIGH\r\n");
    }
    else
    {
        printf("Alert pin is configured as ACTIVE-LOW\r\n");
    }
    return polarity;
}
\end{lstlisting}

%--- SET/GET ALERT PIN (CNVR) ---
\subsubsection{Configuración de Alerta por Conversión Completa}
Asigna la conmutación física del pin ALERT para notificar externamente al microcontrolador cada vez que el ADC del sensor termina una conversión analógica a digital.

\begin{itemize}
	\item \textbf{Funcionamiento Interno:}
		Modifica el bit 10 (\texttt{CNVR}) del registro de máscara mediante \texttt{INA236\_CNVR\_Mask} y la posición \texttt{INA236\_CNVR\_Pos} (\texttt{10U}). La función de consulta lee el registro, extrae el bit mediante desplazamiento lógico a la derecha por 10 posiciones e imprime un aviso en consola informando al usuario sobre la configuración actual del pin de alerta.
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
void INA236_SetAlertPin(INA236_HandleTypeDef *ina236, INA236_ConvRdy_TypeDef cnvr)
{
    uint16_t regValue = INA236_ReadRegister(ina236, INA236_MASK_ENABLE_REGISTER);
    regValue &= ~INA236_CNVR_Mask;
    regValue |= (cnvr << INA236_CNVR_Pos) & INA236_CNVR_Mask;
    INA236_WriteRegister(ina236, INA236_MASK_ENABLE_REGISTER, regValue);
}

INA236_ConvRdy_TypeDef INA236_GetAlertPin(INA236_HandleTypeDef *ina236)
{
    uint16_t regValue = INA236_ReadRegister(ina236, INA236_MASK_ENABLE_REGISTER);
    if (regValue == 0xFFFF)
    {
        printf("ERROR - ALERT pin does not assert on conversion ready\r\n");
        return INA236_CONV_READY_ALERT_DISABLE;
    }
    INA236_ConvRdy_TypeDef alert = (INA236_ConvRdy_TypeDef)((regValue & INA236_CNVR) >> INA236_CNVR_Pos);
    if (alert == INA236_CONV_READY_ALERT_ENABLE)
    {
        printf("ALERT pin asserts when conversion is ready\r\n");
    }
    else
    {
        printf("ALERT pin does not assert on conversion ready\r\n");
    }
    return alert;
}
\end{lstlisting}

\subsubsection{Funciones de Selección de Límites de Alerta}
Estas funciones configuran los bits superiores de habilitación en el registro de máscara para que el chip compare internamente y dispare el pin físico de alerta si las lecturas de entrada superan los límites configurados por el firmware en el registro de límite de alerta \texttt{ALERT\_LIMIT} (\texttt{0x07}).

\begin{itemize}
	\item \textbf{Lógicas de Alerta de Límites:}
		\begin{description}
			\item[\texttt{SetPowerOverLimit}:] Activa el bit 11 (\texttt{POL}) para disparar si la potencia de la carga excede el límite.
			\item[\texttt{SetBusUnderLimit}:] Configura el bit 12 (\texttt{BUL}) para conmutar si la tensión del bus cae por debajo del umbral de seguridad.
			\item[\texttt{SetBusOverLimit}:] Programa el bit 13 (\texttt{BOL}) para conmutar si la tensión del bus supera el límite de seguridad.
			\item[\texttt{SetShuntUnderLimit}:] Asigna el bit 14 (\texttt{SUL}) para alertar si la caída de tensión en el shunt es inferior a la especificada.
			\item[\texttt{SetShuntOverLimit}:] Asigna el bit 15 (\texttt{SOL}) para alertar si la caída de tensión en el shunt excede el umbral seguro.
		\end{description}
\end{itemize}

void INA236_SetPowerOverLimit(INA236_HandleTypeDef *ina236, INA236_PowerAlert_TypeDef powerAlert)

\subsection{Funciones de Lectura de Diagnóstico e Identificación}
Estas funciones monitorean el estado de las banderas internas de conversión de datos por software (polleo de estados) y permiten comprobar la validez del hardware leyendo los códigos únicos grabados en fábrica.

%--- DIAGNOSTIC FLAGS ---
\subsubsection{Lectura de Banderas de Conversión y Desbordamiento}
Permiten consultar si existen datos listos de tensión/corriente para lectura o si ha ocurrido un error matemático crítico.

\begin{itemize}
	\item \textbf{Funcionamiento Interno:}
		\begin{description}
			\item[\texttt{INA236\_IsConversionReady}:] Consulta el estado de actualización del ADC aislando el bit 3 (\texttt{CVRF}) del registro de máscara. Devuelve \texttt{1} si hay una nueva conversión lista para ser leída. La bandera se borra automáticamente al leer el registro o al escribir en configuración.
			\item[\texttt{INA236\_IsMathOverflowReady}:] Consulta si ocurrió una saturación analógica o desbordamiento en el coprocesador matemático del chip leyendo el bit 2 (\texttt{OVF}). Si el bit es \texttt{1}, las lecturas matemáticas internas de corriente y potencia no son válidas por desbordamiento y deben descartarse.
		\end{description}
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
_Bool INA236_IsConversionReady(INA236_HandleTypeDef *ina236)
{
    uint16_t regValue = INA236_ReadRegister(ina236, INA236_MASK_ENABLE_REGISTER);
    if (regValue == 0xFFFF)
    {
        return 0;
    }
    if ((regValue & INA236_CVRF) != 0U)
    {
        return 1; // Conversion Ready
    }
    return 0;
}

uint8_t INA236_IsMathOverflowReady(INA236_HandleTypeDef *ina236)
{
    uint16_t regValue = INA236_ReadRegister(ina236, INA236_MASK_ENABLE_REGISTER);
    if (regValue == 0xFFFF)
    {
        return 0;
    }
    if ((regValue & INA236_OVF) != 0U)
    {
        return 1; // Math Overflow occurred
    }
    return 0;
}
\end{lstlisting}

\subsection{Funciones de Instrumentación y Medida Física}
Estas funciones implementan las conversiones matemáticas definitivas. Extraen los datos crudos firmados en enteros de dos complementos desde los registros de medida y los escalan utilizando multiplicadores en punto flotante (\texttt{double}) para entregárselos a la aplicación.

%--- GET SHUNT VOLTAGE ---
\subsubsection{Lectura de Tensión de Shunt}
Devuelve el voltaje diferencial en bornes del shunt medido por el ADC, expresado en millivolts (mV).

\begin{itemize}
	\item \textbf{Funcionamiento Interno:}
		Lee el registro \texttt{SHUNT\_VOLTAGE\_REGISTER} (\texttt{0x01}). Convierte el entero sin signo recibido a un tipo de entero de 16 bits firmado de manera explícita:
		\begin{equation}
			\text{rawValue} = (\text{int16\_t})\text{regValue}
		\end{equation}
		Esto preserva de forma nativa la firma bidireccional de dos complementos de la corriente en la resistencia (los valores negativos indican flujo inverso de carga). Finalmente, calcula los millivolts reales multiplicando por la resolución física configurada en el ADC (\texttt{\_resolution} en Volts) y por un factor de escala de 1000.0:
		\begin{equation}
			\text{Retorno} = \text{rawValue} \times \text{ina236}\rightarrow\text{\_resolution} \times 1000.0
		\end{equation}
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
double INA236_GetShuntVoltage_mV(INA236_HandleTypeDef *ina236)
{
    uint16_t regValue = INA236_ReadRegister(ina236, INA236_SHUNT_VOLTAGE_REGISTER);
    int16_t rawValue = (int16_t)regValue;
    return (double)rawValue * ina236->_resolution * 1000.0;
}
\end{lstlisting}

%--- GET BUS VOLTAGE ---
\subsubsection{Lectura de Tensión de Bus}
Devuelve el voltaje real del bus de alimentación medido en la terminal de carga, expresado en Volts (V).

\begin{itemize}
	\item \textbf{Funcionamiento Interno:}
		Lee el registro \texttt{BUS\_VOLTAGE\_REGISTER} (\texttt{0x02}). Al tratarse de una tensión de bus de alimentación que no puede ser negativa en polaridad normal, el valor se procesa como entero sin signo. Para evitar que un error de comunicación I2C (que devuelve \texttt{0xFFFF}) sea interpretado como una lectura válida gigante de $104.8\text{ V}$, se valida si \texttt{regValue == 0xFFFF} para retornar en ese caso \texttt{0.0}. De lo contrario, escala el valor de los LSB leídos por el paso constante interno de hardware del integrado de $1.6\text{ mV}$ ($0.0016\text{ V}$):
		\begin{equation}
			V_{\text{bus}}\text{ [V]} = \text{regValue} \times 0.0016
		\end{equation}
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
double INA236_GetBusVoltage(INA236_HandleTypeDef *ina236)
{   
    uint16_t regValue = INA236_ReadRegister(ina236, INA236_BUS_VOLTAGE_REGISTER);
    if (regValue == 0xFFFF)
    {
        return 0.0;
    }
    return (double)regValue * 0.0016;
}
\end{lstlisting}

%--- GET CURRENT ---
\subsubsection{Lectura de Corriente}
Devuelve la corriente eléctrica instantánea calculada que circula a través de la carga, expresada en Amperes (A).

\begin{itemize}
	\item \textbf{Funcionamiento Interno:}
		Lee el registro interno \texttt{CURRENT\_REGISTER} (\texttt{0x04}). Si ocurre un fallo en el bus de datos (retorno \texttt{0xFFFF}), devuelve de forma segura \texttt{0.0}. En caso de éxito, convierte la lectura a un entero firmado de 16 bits de dos complementos para conservar el signo del flujo, y multiplica el valor crudo por el LSB de corriente seleccionado y redondeado en la fase de calibración:
		\begin{equation}
			I\text{ [A]} = (\text{int16\_t})\text{regValue} \times \text{ina236}\rightarrow\text{\_currentLsb}
		\end{equation}
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
double INA236_GetCurrent_A(INA236_HandleTypeDef *ina236)
{
    uint16_t regValue = INA236_ReadRegister(ina236, INA236_CURRENT_REGISTER);
    if (regValue == 0xFFFF) return 0.0;
    return (double)((int16_t)regValue) * ina236->_currentLsb;
}
\end{lstlisting}

%--- GET POWER ---
\subsubsection{Lectura de Potencia Consumida}
Devuelve la potencia eléctrica disipada en la carga por el sistema, expresada en Watts (W).

\begin{itemize}
	\item \textbf{Funcionamiento Interno:}
		Lee el registro \texttt{POWER\_REGISTER} (\texttt{0x03}). La potencia siempre se almacena en enteros positivos unipolar sin signo. La función comprueba errores I2C y, en condiciones normales, escala el registro leído aplicando el multiplicador físico definido por la arquitectura de silicio del INA236: el paso de la potencia es equivalente a $32$ veces el LSB de la corriente:
		\begin{equation}
			P\text{ [W]} = \text{regValue} \times (32.0 \times \text{ina236}\rightarrow\text{\_currentLsb})
		\end{equation}
\end{itemize}

\begin{lstlisting}[style=CMakeStyle]
double INA236_GetPower_W(INA236_HandleTypeDef *ina236)
{
    uint16_t regValue = INA236_ReadRegister(ina236, INA236_POWER_REGISTER);
    if (regValue == 0xFFFF) return 0.0;
    // Power LSB is fixed at 32 * Current_LSB
    return (double)regValue * (32.0 * ina236->_currentLsb);
}
\end{lstlisting}

\section{Integración de la Librería en CMake y Soporte de Punto Flotante}

Para utilizar de forma permanente la librería del sensor INA236 en un desarrollo basado en CMake en Visual Studio Code, se deben integrar sus fuentes de forma explícita y configurar el soporte físico para punto flotante en el microcontrolador.

\subsection{Modificación del archivo \texttt{CMakeLists.txt}}
Las cabeceras y fuentes agregadas a mano deben vincularse en las secciones correspondientes de la siguiente manera:
\begin{lstlisting}[style=CMakeStyle]
# Add sources to executable
target_sources(${CMAKE_PROJECT_NAME} PRIVATE
    # Add user sources here
    Core/INA236/Src/INA236.c
)

# Add include paths
target_include_directories(${CMAKE_PROJECT_NAME} PRIVATE
    # Add user defined include paths
    Core/INA236/Inc
)
\end{lstlisting}

\subsection{Habilitación de Punto Flotante en la Toolchain (Newlib-nano)}
Por defecto, la compilación de proyectos embebidos para STM32 utiliza la versión de librería estándar ligera denominada \textbf{Newlib-nano}. Para ahorrar memoria Flash y RAM en microcontroladores con recursos críticos, Newlib-nano deshabilita de fábrica el soporte para dar formato de salida a variables de tipo flotante (\texttt{float} o \texttt{double}) mediante llamadas a funciones clásicas como \texttt{sprintf} o \texttt{printf}.

Si intenta imprimir el valor decimal devuelto por \texttt{INA236\_GetCurrent\_A()} con la sentencia:
\begin{lstlisting}[style=CMakeStyle]
char msgBuffer[100];
sprintf(msgBuffer, "Corriente: %.3f A\r\n", currentVal);
\end{lstlisting}
El búfer resultante omitirá los decimales o mostrará un signo de interrogación vacío en la terminal serie.

Para forzar al compilador de GCC a enlazar las librerías matemáticas completas y permitir la conversión de datos decimales, se deben activar explícitamente las opciones de enlazador en el archivo \texttt{CMakeLists.txt}. Busque la sección del bloque \texttt{target\_link\_options} y agregue las banderas correspondientes como se muestra a continuación:

\begin{lstlisting}[style=CMakeStyle]
target_link_options(${CMAKE_PROJECT_NAME} PRIVATE
    -u_printf_float   # Habilita el soporte para formatear flotantes en sprintf y printf
    -u_scanf_float    # Habilita el soporte para recibir flotantes en sscanf y scanf
)
\end{lstlisting}

\begin{InfoBox}{Costo en Memoria de las Banderas de Flotantes}
	Activar estas dos directivas matemáticas agrega aproximadamente entre $8\text{ KB}$ y $12\text{ KB}$ adicionales al tamaño final del archivo binario compilado en la Flash del procesador. Es muy importante comprobar el uso de recursos desde la pestaña \textbf{STM32 Build Analyzer} de VS Code para asegurar que no exceda los límites de memoria física del procesador.
\end{InfoBox}
