\section*{Resumen}
Este manual técnico describe los pasos y consideraciones indispensables para configurar Visual Studio Code como el editor de texto principal para LaTeX en el sistema operativo Windows. Se detallan las herramientas necesarias (VS Code, MiKTeX, Strawberry Perl), la instalación de la extensión LaTeX Workshop y el flujo de preparación del entorno de compilación en tiempo real. Adicionalmente, se explica cómo diagnosticar y resolver el bloqueo común del compilador \texttt{latexmk} derivado de la ausencia de un intérprete de Perl, ofreciendo dos soluciones prácticas (la instalación de Perl y la reconfiguración directa a \texttt{pdflatex}). Por último, se presenta un análisis comparativo y recomendaciones de uso entre los motores de compilación \texttt{latexmk} y \texttt{pdflatex}.

\section{Introducción al Entorno de LaTeX en VS Code}

Escribir documentos científicos y técnicos en LaTeX tradicionalmente requería el uso de editores dedicados como TeXstudio o TeXmaker. No obstante, en el desarrollo contemporáneo, la integración de LaTeX en editores multiplataforma y multipropósito como \textbf{Visual Studio Code} (VS Code) se ha consolidado como la opción preferida por los desarrolladores. 

La combinación de VS Code con extensiones especializadas proporciona ventajas críticas de productividad, entre las que destacan el autocompletado inteligente de comandos, el formateo en tiempo real, el resaltado de errores en el código fuente, la sincronización de visualización (SyncTeX) y una visualización integrada del PDF final lado a lado con el código fuente.

\section{Prerrequisitos de Aplicaciones y Extensiones}

Para habilitar un espacio de trabajo completo y funcional de LaTeX en VS Code, es necesario instalar tres componentes principales en tu sistema operativo:

\subsection{Aplicaciones necesarias (Software del sistema)}
\begin{enumerate}
	\item \textbf{Visual Studio Code:} El entorno o editor de texto base. Descárgalo de su página oficial: \href{https://code.visualstudio.com/}{code.visualstudio.com}.
	\item \textbf{Distribución de LaTeX (MiKTeX o TeX Live):} Los editores de texto no pueden compilar archivos por sí solos; necesitan los binarios y paquetes del compilador LaTeX. Para Windows, la distribución recomendada es \textbf{MiKTeX} por su sistema de instalación automática de paquetes bajo demanda. Descárgala desde \href{https://miktex.org/}{miktex.org} e instálala en tu sistema.
	\item \textbf{Intérprete de Perl (Opcional, pero recomendado):} Requerido únicamente si deseas usar la herramienta de automatización por defecto \texttt{latexmk}. En Windows, la distribución recomendada es \textbf{Strawberry Perl}. Se puede descargar desde \href{https://strawberryperl.com/}{strawberryperl.com} o instalar directamente a través de la terminal.
\end{enumerate}

\subsection{Extensiones necesarias en VS Code}
Abre Visual Studio Code, accede al panel de extensiones (\keys{Ctrl + Shift + X}), busca e instala la siguiente extensión imprescindible:
\begin{itemize}
	\item \textbf{LaTeX Workshop} (creada por James Yu): Esta extensión dota a VS Code de un entorno integrado de LaTeX, incluyendo compilación automática al guardar, visualizador de PDF interactivo, herramientas de autocompletado para bibliografía, etiquetas y citas, y visualización de fórmulas matemáticas al pasar el cursor.
\end{itemize}

\section{Preparación y Uso del Entorno de Trabajo}

Una vez instalado el compilador (MiKTeX) y la extensión en el editor (LaTeX Workshop), el entorno está listo para ser utilizado:

\subsection{Visualización en tiempo real (Visualizador de PDF)}
LaTeX Workshop incluye un lector de PDF integrado que sincroniza automáticamente los cambios realizados en el archivo \texttt{.tex} cada vez que guardas el documento:
\begin{itemize}
	\item Abre tu archivo \texttt{.tex} en el editor.
	\item Compila el archivo por primera vez usando la combinación \keys{Ctrl + Alt + B} (o haz clic en el icono verde de "Play" arriba a la derecha).
	\item Para abrir la vista previa del PDF en una pestaña de VS Code al lado de tu código, presiona la combinación \keys{Ctrl + Alt + V} (o haz clic en el icono del PDF en la esquina superior derecha).
\end{itemize}

\subsection{Sincronización Bidireccional (SyncTeX)}
SyncTeX es una herramienta que conecta de forma exacta la posición de tu cursor en el código fuente con la página correspondiente en el visualizador PDF y viceversa:
\begin{itemize}
	\item \textbf{De código a PDF (Forward Sync):} Si estás editando un párrafo de código y deseas ver cómo se renderiza en el PDF, mantén presionada la tecla \keys{Ctrl} y haz clic izquierdo con el ratón en la línea de código correspondiente (o presiona \keys{Ctrl + Alt + J}). El visor de PDF se desplazará de inmediato y resaltará temporalmente la zona correspondiente en el PDF.
	\item \textbf{De PDF a código (Backward Sync):} Si estás observando el visor PDF de VS Code y notas un error en un párrafo, mantén presionada la tecla \keys{Ctrl} y haz doble clic izquierdo sobre el texto en el PDF. El cursor de tu editor se trasladará de inmediato a la línea exacta de código fuente.
\end{itemize}

\section{Diagnóstico y Resolución del Error de `latexmk`}

Al intentar compilar un proyecto recién creado en VS Code, es muy común encontrarse con el siguiente mensaje de error en la esquina inferior derecha o en los logs de salida del editor:

\begin{lstlisting}[language=plaintext, style=miestiloclaro]
Sorry, but latexmk did not succeed for the following reason:
  MiKTeX could not find the script engine 'perl' which is required to execute 'latexmk'.
Remedy:
  Make sure 'perl' is installed on your system.
The log file hopefully contains the information to get MiKTeX going again:
  C:\Users\casti\AppData\Local\MiKTeX\miktex\log\latexmk.log
For more information, visit: https://miktex.org/kb/fix-script-engine-not-found
\end{lstlisting}

\subsection{¿A qué se debe este error?}
Por defecto, la extensión \textbf{LaTeX Workshop} utiliza el motor de automatización llamado \texttt{latexmk} para compilar los documentos. Dado que \texttt{latexmk} es un script escrito en el lenguaje de programación \textbf{Perl}, necesita un intérprete de Perl instalado en el sistema operativo para poder ejecutarse. Windows no incluye Perl por defecto, por lo que la compilación se bloquea de inmediato.

Dispones de dos soluciones distintas para corregir este comportamiento:

\subsection{Solución A: Instalar Perl en Windows (Recomendada)}
Esta es la solución definitiva si deseas conservar el flujo inteligente de compilación automática que proporciona `latexmk`.

\begin{enumerate}
	\item \textbf{Instalar mediante consola (Método rápido):} Abre una terminal de PowerShell o CMD y ejecuta el siguiente comando oficial del administrador de paquetes de Windows:
	\begin{lstlisting}[language=DOS]
	winget install StrawberryPerl.StrawberryPerl
	\end{lstlisting}
	\item \textbf{Instalar mediante la web:} Si prefieres el método manual, ve al sitio oficial \href{https://strawberryperl.com/}{strawberryperl.com}, descarga el instalador con extensión \texttt{.msi} para tu arquitectura y ejecútalo en Windows.
	\item \textbf{Reinicio crítico:} Una vez finalizada la instalación, \textbf{debes cerrar completamente Visual Studio Code y volverlo a abrir}. Esto recarga las variables de entorno del sistema operativo, permitiendo que VS Code reconozca la ruta de Perl en el \texttt{PATH}.
\end{enumerate}

\subsection{Solución B: Cambiar el motor a `pdflatex` directamente (Sin instalar Perl)}
Si no deseas instalar aplicaciones adicionales en tu máquina como Perl y prefieres un entorno portátil o simple, puedes configurar la extensión para que use el compilador `pdflatex` de MiKTeX directamente en lugar de `latexmk`:

\begin{enumerate}
	\item En VS Code, abre la paleta de comandos globales presionando \keys{Ctrl + Shift + P}.
	\item Escribe la directiva \texttt{Preferences: Open User Settings (JSON)} y presiona \keys{Enter}.
	\item Dentro de las llaves principales del archivo \texttt{settings.json}, añade las siguientes líneas de configuración:
	
	\begin{lstlisting}[language=json]
{
    "latex-workshop.latex.recipes": [
        {
            "name": "pdflatex -> bibtex -> pdflatex x2",
            "tools": [
                "pdflatex",
                "bibtex",
                "pdflatex",
                "pdflatex"
            ]
        }
    ],
    "latex-workshop.latex.tools": [
        {
            "name": "pdflatex",
            "command": "pdflatex",
            "args": [
                "-synctex=1",
                "-interaction=nonstopmode",
                "-file-line-error",
                "%DOC%"
            ]
        },
        {
            "name": "bibtex",
            "command": "bibtex",
            "args": [
                "%DOCFILE%"
            ]
        }
    ]
}
	\end{lstlisting}
	
	\item Guarda el archivo de configuración con \keys{Ctrl + S} y cierra la pestaña. Al volver a compilar tu archivo LaTeX, se procesará con \texttt{pdflatex} sin pedirte Perl.
\end{enumerate}

\section{Comparativa de Motores: `latexmk` vs `pdflatex`}

Es crucial entender la diferencia técnica entre ambos métodos para elegir la opción adecuada según la complejidad de tus proyectos académicos o profesionales.

\subsection{El motor pdflatex}
\textbf{pdflatex} es el binario básico del compilador. Toma directamente el archivo de texto fuente \texttt{.tex} y lo traduce a código de máquina PDF.
\begin{itemize}
	\item \textbf{Mecanismo:} Compila de manera secuencial y ejecuta exactamente una pasada. 
	\item \textbf{El problema de referencias vacías:} Si agregas una sección nueva, referencias cruzadas (\texttt{\textbackslash label} y \texttt{\textbackslash ref}) o bibliografía (BibTeX), \texttt{pdflatex} no resolverá los enlaces en la primera pasada, dejando marcas en su lugar (signos de interrogación \textbf{??}). Se requiere ejecutar manualmente el comando de 2 a 3 veces consecutivas para actualizar los archivos auxiliares (\texttt{.aux}, \texttt{.toc}, \texttt{.bbl}) y que los enlaces aparezcan correctamente.
\end{itemize}

\subsection{El script latexmk}
\textbf{latexmk} es un envoltorio o script de automatización inteligente escrito en Perl que gestiona el proceso completo de compilación de forma autónoma.
\begin{itemize}
	\item \textbf{Mecanismo:} Analiza el archivo fuente y examina el contenido de los archivos auxiliares generados en cada compilación. \texttt{latexmk} detecta si hay dependencias sin resolver o índices que necesitan actualización, y ejecuta internamente \texttt{pdflatex}, \texttt{bibtex} o \texttt{makeindex} la cantidad exacta de veces necesarias de forma totalmente transparente para el usuario.
\end{itemize}

\subsection{Conclusión: ¿Cuál de los dos es mejor utilizar?}
\begin{itemize}
	\item \textbf{Usa \texttt{latexmk} (Recomendado) si:} Trabajas en proyectos grandes de múltiples páginas (libros, tesis o reportes extensos) que cuenten con una amplia cantidad de referencias cruzadas, listas de imágenes, índices alfabéticos y bases de datos bibliográficas complejas (\texttt{.bib}). La automatización te ahorrará tiempo y evitará PDFs rotos o incompletos.
	\item \textbf{Usa \texttt{pdflatex} si:} Estás realizando documentos simples, de una o pocas páginas (como cartas, tareas rápidas, CVs sencillos), o si estás trabajando en computadoras con restricciones de seguridad donde no tienes permisos para instalar programas adicionales como Perl.
\end{itemize}
